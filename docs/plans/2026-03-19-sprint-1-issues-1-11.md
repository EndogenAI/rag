## Definition of Done (Sprint 1)
- Issues #1-#7 are either completed or have explicit, justified carry-over notes.
- Issues #9-#11 produce written decisions that are reflected in implementation artifacts.
- Core path works: local index -> MCP query -> usable retrieval output in VS Code workflow.
- CI includes at least one retrieval smoke signal and basic test/lint gates.

## Key Risks
1. Scope drift from research into implementation details without gating.
2. Index/chunking contract churn causing rework in MCP and evaluation layers.
3. Delayed security/eval decisions forcing late redesign.

## First Actions
1. Triage issues #9-#11 into a single research kickoff checklist.
2. Start Phase 1 research pass and log outputs against implementation constraints.
3. Begin Phase 2 only after Review Gate A sign-off.
