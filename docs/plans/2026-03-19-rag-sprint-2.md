## PR Workflow Constraints (Sprint 2)
- Every PR must include a detailed body with explicit issue/artifact traceability.
- Every PR that closes issues must include Closes/Fixes/Resolves lines in body text.
- Run local PR-close guard before push:
  - uv run python scripts/check_current_pr_closes.py
- CI must be green before requesting review.
- On reviews, run full loop:
  - triage comments
  - apply fixes
  - post batch replies
  - resolve all addressed threads
  - verify resolved thread state
